# Mary Workflow Course Learning Profile

Use this prompt with `skills/lecture-learning/SKILL.md` as the governing content contract.

## Entry

The user's request is the current learning scope. Confirm or infer only the minimum needed to identify the course, Lecture/topic, available slides, transcript status, existing Course Hub/Lecture page, and today's bounded learning block. Surface every inference before freezing a plan.

If the phase is `FINISHED`, tell the user to run `/mw-cycle` first. If the phase is `PLANNING`, use the normal `/mw-plan` interview and turn the request into 1–3 bounded milestones. If the phase is `PLANNED`, present the exact plan and let `/mw-run` consume the grant. If execution is active, work only on the current milestone.

## Execution Route

1. Inspect Course Hub, child pages, syllabus/schedule, and Source Index.
2. Classify material and run the Slide to Lecture foundation.
3. Reuse/create the raw recording child page in the exact location and naming required by the Lecture skill.
4. When a transcript exists, correct against slides and fuse only recording-only additions with `🎙` markers.
5. Review parentage, ordering, formulas, images, deadlines, uncertainty markers, summary, and next action.

Treat Notion pages and calendar records as deliverables. Keep raw transcript text unchanged. Record missing inputs as a gate instead of fabricating them.

## Acceptance Evidence

The completion action must name the affected Notion pages/URLs or local source files and state which Lecture-learning checklist items passed, which inputs were unavailable, and what the next single learning action is.
